# 00 — Complexity Gate：RAG 操作複雜度評估

> 每次 RAG 操作前必須先經過複雜度評估，決定 Lite / Standard / Full 模式。

## 五題計分

| # | 問題 | 0 分 | 1 分 | 2 分 |
|---|------|------|------|------|
| 1 | 涉及多少 namespace？ | 1 個 | 2-3 個 | 4+ 或全域 |
| 2 | 有存取權限變動嗎？ | 無 | 修改既有權限 | 新增 namespace 權限 |
| 3 | 需要 re-embed 嗎？ | 不需要 | 部分文件 | 全部文件 |
| 4 | 涉及外部系統整合嗎？ | 否 | 讀取外部 API | 寫入外部系統 |
| 5 | 影響多少用戶？ | <10 人 | 10-100 人 | 100+ 或全公司 |

## 模式判定

| 總分 | 模式 | 必要閘門 |
|------|------|---------|
| 0-2 | Lite | 00 + 03 |
| 3-5 | Standard | 00 + 01 + 02 + 03 + 04 |
| 6+ | Full | 全部 + ADR + HITL 確認 |

## 紀錄格式

```
Complexity Score: [N] / 10
Mode: [Lite | Standard | Full]
Rationale: [簡述各題分數]
```
