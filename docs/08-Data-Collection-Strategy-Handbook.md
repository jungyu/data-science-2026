# 📦 資料收集與整理
## ——寫給真正想搞懂資料的你

---

## 🎖️ 開場故事：一個排長，一間神秘小房間

越南，1962 年。叢林裡的夜晚屬於越共。

年輕的科林·鮑爾（Colin Powell）中尉，剛從西點軍校畢業沒多久，就被派到越南擔任步兵排長。他帶著一群士兵，每天在潮濕的叢林裡行軍、伏擊、反伏擊。

幾週下來，鮑爾和他的弟兄們用生命學到了一件事：

```
越共是夜行動物。
白天：消失在叢林裡，不見蹤影。
夜晚：無聲無息地移動、攻擊、補給。

這不是假設，這是用血換來的情報。
```

某天，鮑爾奉命回到師部（Division HQ）彙報任務。  
在那棟乾淨、有冷氣的建築裡，他注意到一間神秘的小房間。  
房間外堆滿了文件，裡面的參謀每天忙著整理、分析、建模。  
大量的報告、數字、統計表格，像河流一樣流進那個房間。

鮑爾好奇地靠近，問了一個問題：  
**「你們在做什麼？」**

參謀驕傲地回答：  
**「我們在分析越共的活動模式，預計再幾個月就能得出結論。」**

鮑爾沉默了一秒，然後說：

> **「我們在叢林裡待了三週就知道答案了。  
> 他們晝伏夜出。  
> 你們的資料，告訴你們的，是同一件事。」**

---

然後他轉身走出了那個房間。

---

### 🧠 這個故事在說什麼？

這不是在嘲笑參謀們不聰明。他們很聰明，他們非常努力。  
問題出在別的地方：

```
錯誤一：收集了大量資料，卻沒有問「這些資料從哪裡來？」
         → 報告是由誰填的？在什麼情況下填的？

錯誤二：沒有定義「什麼才算有用的資訊」
         → 分析了幾個月，結論卻是前線三週就知道的事

錯誤三：把「資料豐富」誤認為「資料正確」
         → 有更多數字，不代表更接近真相
```

鮑爾後來成為美國國務卿，他在回憶錄中反覆強調一個原則：  
**「收集資料之前，先搞清楚你要回答的問題是什麼。」**

這，就是這本手冊的起點。

---

> **「垃圾進，垃圾出。」**  
> *(Garbage In, Garbage Out)*  
> 這不只是一句老話，這是每一個曾經被錯誤資料坑過的人，  
> 用慘痛教訓刻出來的誡律。

# 第一站：資料偏誤 🎭
## 你的資料在說謊，而且說得很有自信

---

### 🔴 停！先想一個問題

> 你想研究「台灣人的消費習慣」。  
> 你站在信義區某百貨公司門口，訪問了 500 個人。  
> 你覺得這份資料可以代表「台灣人」嗎？

**答案是：不行。** 這就是偏誤。

---

### 什麼是偏誤 (Bias)？

偏誤是指：**資料無法真實代表你想研究的母體，產生系統性的誤差。**

關鍵詞是「**系統性**」——它不是隨機的運氣不好，它是一個穩定、持續在扭曲你資料的力量。

---

### 🃏 偏誤大家族——三張王牌

---

#### 🂡 A 牌：抽樣偏誤 (Sampling Bias)

```
┌─────────────────────────────────────┐
│  定義：你的樣本，根本不代表整體母體  │
└─────────────────────────────────────┘
```

**場景還原：**

> 🏬 百貨公司門口調查消費習慣  
> → 你問到的全是「願意出門逛街」的人  
> → 宅在家用 Shopee 的電商族完全消失了  
> → 你的結論：「台灣人喜歡實體購物」（錯！）

---

### 📰 真實故事：史上最大規模的民調，史上最丟臉的預測

美國，1936 年。總統大選前夕。

《文學文摘》雜誌（*The Literary Digest*）是當時美國最權威的媒體之一。  
他們做了一件前所未有的事：  
寄出了 **1,000 萬份**問卷，最終回收 **240 萬份**有效回覆。

這是史上規模最大的民調。他們信心滿滿地宣布：

```
預測結果：共和黨候選人蘭登（Alf Landon）將以 57% 比 43% 大勝。
          羅斯福（Franklin Roosevelt）將連任失敗。
```

選舉結果揭曉：

```
實際結果：羅斯福以 61% 比 37% 壓倒性大勝。
          蘭登只贏了 2 個州。
```

**240 萬份問卷。全錯。**

---

**原因是什麼？**

```
他們的收件名單來自：
  ① 汽車登記資料庫
  ② 電話號碼簿

1936 年，美國正值大蕭條。
能擁有汽車或電話的人，都是中上階層。
中下階層——那些在大蕭條中最支持羅斯福新政的人——
根本不在名單裡。

240 萬份問卷代表的，是同一個族群的 240 萬個聲音。
而不是美國。
```

> 💀 **後記：** 《文學文摘》雜誌在這次預測失敗後聲譽崩潰，  
> 兩年後停刊，永久走入歷史。  
> 一份問卷，毀了一本雜誌。

**這個故事的教訓：**

樣本數量不等於樣本品質。  
**1,000 個代表性樣本，勝過 100 萬個有偏誤的樣本。**

---

**處理原則：**

| 方法 | 說明 |
|------|------|
| 🎲 隨機抽樣 | 每個人被選到的機率相等 |
| 📊 分層抽樣 | 按比例從各子群體取樣（年齡、性別、地區） |
| 🔀 多管道收集 | 線上 + 線下 + 電話，別只守一個管道 |

---

#### 🂡 K 牌：標籤偏誤 (Label Bias)

```
┌─────────────────────────────────────┐
│  定義：人工標記時，帶入了主觀偏見   │
└─────────────────────────────────────┘
```

**場景還原：**

> 🏥 兩位醫生標記同一批醫學影像  
> 醫生A：「這是良性腫瘤」  
> 醫生B：「這是惡性腫瘤」  
> 
> 你的 AI 學到的到底是什麼？  
> → 它學到了**醫生的偏見**，不是事實。

---

### ⚖️ 真實故事：決定你是否入獄的演算法

美國，2016 年。法院不只由法官定罪，還有一套 AI 系統在幫忙評估。

**COMPAS**（矯正罪犯管理概況）系統被廣泛用於美國各州，  
協助法官評估被告的「再犯風險評分」——分數越高，越可能被判重刑或拒絕假釋。

ProPublica 新聞調查團隊拿到了資料，做了一件簡單的分析：

```
分析對象：佛羅里達州 7,000 名被告，追蹤兩年後的實際再犯情況

結果發現：

被標記為「高風險」但實際上沒有再犯的人：
  白人被告：23.5%  ← 錯誤率
  黑人被告：44.9%  ← 錯誤率是白人的 1.9 倍

被標記為「低風險」但實際上再犯的人：
  白人被告：47.7%
  黑人被告：28.0%
```

這個系統對黑人被告有系統性的「誤判為高風險」傾向，  
對白人被告則有「誤判為低風險」的傾向。

**標籤從哪裡來？**

```
COMPAS 的訓練資料來自歷史判決記錄。
而歷史判決記錄，本身就帶著幾十年的司法偏見。

垃圾標籤進去 → 垃圾判斷出來
只是這次的垃圾，決定了一個人能不能回家。
```

**處理原則：**

```
多重審核 + 一致性指標

步驟 1：讓多個標記者獨立標記
步驟 2：計算「標注者間一致性」
         → 常用指標：Cohen's Kappa (κ)
           κ > 0.8 = 高度一致 ✅
           κ < 0.6 = 需要重新審視標準 ⚠️
步驟 3：歧異處開會討論，建立明確標記規範
步驟 4：對不同子群體分別測試標籤的錯誤率（公平性審計）
```

#### 🂡 J 牌：生存者偏誤 (Survivorship Bias)

```
┌─────────────────────────────────────────────┐
│  定義：你只看到「活下來」的資料，           │
│         忽略了「消失」的那些                │
└─────────────────────────────────────────────┘
```

**場景還原：**

> 📚 你研究了 100 位成功創業家的共同特質：  
> → 他們都很努力、充滿熱情、有遠見  
> 
> 你的結論：「努力＋熱情＋遠見 = 創業成功」  
> 
> 問題：**那 1000 位同樣努力、熱情、有遠見但失敗的人去哪了？**  
> → 他們的故事沒人寫，沒人研究，資料庫裡沒有他們。

---

### 💰 真實故事：「績效最佳」基金的騙局

你翻開財經雜誌，看到廣告：  
**「本基金過去五年年均報酬率 18%，遠超大盤！」**

你心動了。

但你不知道的是：

```
2018 年，某資產管理公司旗下有 20 支基金。
2023 年，只剩下 5 支基金還在運作。

那 15 支消失的基金去哪了？
  → 它們虧損嚴重，被悄悄清盤（關閉）了。
  → 一旦清盤，它們的歷史績效記錄從資料庫裡消失。

你看到的「五年績效排行榜」，
只統計了「活到今天的基金」。
那些死掉的，不在榜上。
```

這不是個案。學術研究指出：  
**如果不剔除生存者偏誤，基金的平均回報率會被高估約 1.5% 至 3% 每年。**  
聽起來不多？複利二十年，差距高達數十個百分點。

---

> 💡 **二戰的真實案例：**  
> 統計學家 Abraham Wald 被要求分析返航轟炸機的彈孔，  
> 以決定哪裡需要加厚裝甲。  
> 大家都說「彈孔多的地方要加強」。  
> Wald 說：「**要加強沒有彈孔的地方——因為那些地方中彈的飛機，都沒有飛回來。**」

**處理原則：**
- 🔍 主動尋找「失敗案例」、「消失的樣本」的資料來源
- 📋 確認你的資料集是否有系統性地遺漏某類樣本（被清盤的基金、倒閉的公司、退出的用戶）
- 🗺️ 重建完整的因果圖譜（不只看存活者）
- ❓ 永遠要問：**「我的資料集裡，少了誰？」**

### 🔥 真實案例：AI 招聘系統的翻車現場

```
案例重現
────────────────────────────────────────
公司：某科技巨頭（改編自真實事件）
目標：用 AI 篩選履歷，加速招聘
資料：過去 10 年的招聘決策歷史記錄

問題在哪？

歷史資料中，高階主管 90% 是男性
        ↓
AI 學到了：「男性特徵 → 高評分」
        ↓
結果：女性求職者的分數被自動扣減
────────────────────────────────────────
```

**🛠️ 怎麼修？**

```
解法一：資料平衡 (Re-balancing)
  → 確保各群體在訓練集中比例合理

解法二：剔除敏感特徵
  → 訓練時不餵入「性別」、「姓名」等欄位

解法三：公平性審計 (Fairness Audit)
  → 上線前，對不同群體分別測試模型的預測結果
  → 確認各群體的錯誤率在可接受範圍內
```

> ### ❓ 常見問題 FAQ
>
> **Q：我的資料有偏誤，是不是就不能用了？**  
> A：不是的。有偏誤的資料不等於廢資料。重點是**知道偏誤的存在**，並在分析時加入適當的校正，以及在報告結論時誠實說明資料的侷限性。
>
> **Q：怎麼知道我的資料有沒有偏誤？**  
> A：從「**你的資料是怎麼來的**」開始問。誰收集的？在哪裡收集的？誰的聲音沒有被聽到？

# 第二站：資料缺失 🕳️
## 空格不只是空格，它背後有故事

---

### 🔴 先做個小測驗

> 你的問卷有一題：「請填寫您的月收入」  
> 結果有 30% 的人沒填。  
> 請問：這 30% 的人是隨機沒填的嗎？

**幾乎可以確定：不是。**  
高收入者可能不想透露，低收入者可能感到難堪。  
**缺失本身就是資訊。**

---

### 三種缺失，三種個性

---

#### 🟢 個性一：完全隨機缺失 (MCAR)
**Missing Completely At Random**

```
比喻：感測器突然斷網

特徵：缺失完全是意外，與任何變數無關
好消息：這是最「乾淨」的缺失
壞消息：現實中很少遇到
```

**判斷方式：** 把有缺失和沒缺失的樣本比較，各項特徵分布幾乎相同 → MCAR

---

### 🚀 真實故事：那些「沒問題」的發射資料，害死了七個人

1986 年 1 月 28 日，美國佛羅里達州，氣溫攝氏 2 度。

太空梭「挑戰者號」準備升空。  
工程師們爭論了整整一夜：**低溫會不會影響 O 型環的密封性？**

NASA 的管理層調出了歷次發射的 O 型環損傷記錄，攤開來分析：

```
他們看到的資料：

發射序號   溫度(°C)   O型環損傷？
─────────────────────────────
  STS-2      24°       輕微損傷
  STS-41C    23°       輕微損傷
  STS-51B    27°       輕微損傷
  ...

結論：沒有明顯的溫度與損傷關係。可以發射。
```

**但他們犯了一個致命錯誤。**

他們的資料集裡，**只有發生損傷的發射記錄**。  
那些 O 型環完全正常、沒有損傷的發射——  
沒有被納入分析，因為「沒事幹嘛記錄？」

```
完整資料（包含無損傷的發射）長這樣：

高溫發射（> 20°C）：損傷率低
低溫發射（< 12°C）：損傷率極高

結論完全相反。
```

挑戰者號升空 73 秒後爆炸。七名太空人罹難。

> **這不是 MCAR。這是系統性遺漏。**  
> 但它告訴了我們一個更深的道理：  
> **「沒有問題」的資料不記錄，等於你永遠看不到「沒問題是什麼樣子」。**  
> 沒有對照組，你就沒有分析。

---

#### 🟡 個性二：隨機缺失 (MAR)
**Missing At Random**

```
比喻：女性問卷中，體重欄位空白率較高

特徵：缺失與「其他已知變數」有關，
       但與「缺失值本身」無關
意思：你能從其他欄位「猜到」誰可能會缺失
```

> **舉例：**  
> 受訪者的年齡欄位填了，但體重欄位沒填。  
> 這不是隨機的——年輕女性更傾向於不填體重。  
> 這個缺失與「性別＋年齡」有關。  
> → 這就是 MAR。

---

### 💊 真實故事：消失的病人，失真的藥效

製藥公司正在進行一款新型抗憂鬱藥物的臨床試驗。  
試驗開始時招募了 500 名受試者，隨機分為用藥組與安慰劑組。

十二週後，只剩下 380 人完成了試驗。

**那 120 人去哪了？**

```
退出原因分析：
  ├── 副作用太強烈，無法忍受    → 52 人（幾乎全在用藥組）
  ├── 感覺沒有效果，放棄治療    → 38 人
  └── 其他個人因素              → 30 人

用藥組的退出率：31%
安慰劑組的退出率：9%
```

研究人員發現：**留下來的用藥組受試者，恰好是「對藥物反應較好、副作用較少」的人。**

這個缺失與「受試者的生理敏感性」有關，  
而這個變數部分可以從「體重、年齡、過往用藥史」來預測。

→ **這就是 MAR。**

```
如果你只分析完成試驗的 380 人：
  → 藥效看起來非常好（因為對藥過敏的人都走了）
  → 副作用看起來很低（因為副作用重的人也走了）
  → 你向 FDA 申請上市，然後真實世界的患者吃了之後……
     ……反應完全不一樣。
```

> 這就是為什麼在現代臨床試驗中，  
> 退出樣本的資料一樣要被分析，  
> 並採用「**意向治療分析（Intention-to-Treat Analysis）**」——  
> 不管受試者有沒有完成試驗，都算進最終結果。

---

#### 🔴 個性三：非隨機缺失 (MNAR)
**Missing Not At Random**

```
比喻：高收入者不填收入

特徵：缺失與「缺失值本身」直接相關
危險：你永遠無法從已知資料完全解釋這個缺失
結論：缺失本身就是答案的一部分
```

> **⚠️ 這是最危險的缺失類型。**  
> 如果你假裝它是 MCAR 然後直接刪掉，  
> 你的分析結果會嚴重失真。

---

### 🛠️ 缺失資料處理工具箱

```
┌─────────────────────────────────────────────────────┐
│                  缺失資料處理決策樹                   │
│                                                     │
│   缺失比例 < 5%？                                    │
│       ├── 是 → 🗑️ 刪除法（直接刪除該樣本）            │
│       └── 否 → 繼續往下看                            │
│                                                     │
│   缺失機制是 MCAR 或 MAR？                           │
│       ├── 是 → 📊 填補法 (Imputation)                │
│       └── 否(MNAR) → 🏷️ 標記法 + 模型處理            │
└─────────────────────────────────────────────────────┘
```

---

#### 🗑️ 刪除法
適用時機：缺失比例極低（< 5%），且為 MCAR

```python
# 範例
df.dropna(subset=['target_column'], inplace=True)
```

> ⚠️ **注意：** 刪除有缺失的整列樣本，可能在不知不覺中
> 造成抽樣偏誤！刪之前，先看看被刪的是誰。

---

#### 📊 填補法 (Imputation)

**層級一：統計量填補（快，但粗糙）**

| 資料類型 | 填補方式 |
|----------|----------|
| 連續型（如年齡、收入） | 中位數（不受離群值影響） |
| 連續型（分布對稱） | 平均值 |
| 類別型（如職業、地區） | 眾數 |

**層級二：預測填補（慢，但聰明）**

```
方法一：KNN 填補
  → 找 K 個最相似的樣本，用他們的值來填

方法二：迴歸填補
  → 把缺失欄位當作「應變數」
  → 用其他欄位建一個預測模型
  → 用模型的預測值來填補

方法三：多重填補 (Multiple Imputation)
  → 填補多次，得到多個資料集
  → 分別分析後，合併結論（最嚴謹）
```

---

#### 🏷️ 標記法（把缺失變成特徵）

這個方法的精髓是：**把「不知道」本身當成一種答案。**

```python
# 新增一個欄位，標記是否缺失
df['income_is_missing'] = df['income'].isnull().astype(int)

# 然後再用其他方法填補 income 欄位的空值
df['income'].fillna(df['income'].median(), inplace=True)
```

> 💡 **為什麼這樣做？**  
> 讓模型知道「這筆資料的收入是被填補的，不是原始值」  
> 模型可以學到：「填補標記 = 1 的樣本，行為模式可能不同」

---

> ### ❓ 常見問題 FAQ
>
> **Q：我能不能全部用平均值填補就好？**  
> A：可以，但代價是：你會壓縮資料的變異性，讓所有人都變得「平均」。這會低估模型的不確定性。至少考慮用中位數，或直接上預測填補。
>
> **Q：MNAR 的缺失到底怎麼處理？**  
> A：沒有完美解法。最誠實的做法是：(1) 在報告中說明這個限制；(2) 用標記法讓模型感知缺失的存在；(3) 如果可能，設計更好的資料收集機制，從源頭解決問題。

# 第三站：不可量測性 🌫️
## 你想研究的東西，根本沒辦法直接測量

---

### 🔴 核心困境

> 你想預測「用戶黏性」  
> 但「黏性」是什麼？  
> 你要去哪裡找一個叫做「黏性」的數字？

**現實中，最重要的東西，往往最難量化：**
- 品牌忠誠度
- 員工士氣
- 用戶潛在動機
- 房屋裝潢品質
- 服務體驗滿意度

---

### 🧰 三種量化「看不見的東西」的武器

---

#### ⚔️ 武器一：代理指標 (Proxy Variables)

**核心思想：** 找一個「可以觀察」的量，來代表「無法直接觀察」的量。

---

### 🦠 真實故事：Google 以為自己比 CDC 更懂流感

2008 年，Google 發表了一篇轟動學術界的論文：  
**《用搜尋查詢詞彙預測流感疫情》**

邏輯非常聰明：

```
直接測量「流感擴散程度」→ 困難！
  需要等病患就醫 → 等醫院彙報 → 等 CDC 統計 → 延遲兩週

代理指標：Google 搜尋關鍵字
  「流感症狀」、「發燒怎麼辦」、「退燒藥」的搜尋量
  → 即時！
  → 完全不需要等人去看醫生

2008 年的測試：預測結果與 CDC 資料相關係數高達 0.97
```

全世界的媒體都說：**「Google 打敗了 CDC！大數據萬歲！」**

然後——

```
2013 年流感季：Google Flu Trends 預測的感染人數
               是 CDC 實際數據的兩倍。

完全失準。
```

**為什麼代理指標失效了？**

```
原因一：「查詢行為」改變了，不代表「生病人數」改變
  → 媒體大量報導流感新聞 → 沒生病的人也去搜尋 → 搜尋量暴增

原因二：Google 不斷更新搜尋演算法與自動補全功能
  → 輸入「頭痛」時，Google 自動補全「流感頭痛」
  → 點擊率上升，不是因為更多人生病

原因三：2009 年 H1N1 流感大爆發的媒體效應
  → 訓練資料裡的關係，在新的媒體環境下不再成立
```

> **代理指標的核心警告：**  
> 代理指標在「你訓練它的世界」裡有效。  
> 一旦世界改變，它可能悄悄失效，但它不會告訴你。  
> **永遠要持續驗證你的代理指標是否還在追蹤你真正想測量的東西。**

---

```
真正想測量的             代理指標
─────────────────────────────────────────
「房屋裝潢品質」   →  實價登錄價格 ÷ 同地段平均單價
「學生學習成效」   →  出席率 × 作業繳交率 × 小考平均分
「品牌好感度」     →  提及品牌的社群貼文正面比例
「道路安全程度」   →  每公里事故發生率
「流感擴散程度」   →  Google 搜尋量（⚠️ 需持續校驗）
```

---

#### ⚔️ 武器二：特徵工程與複合指標

**核心思想：** 把多個量化指標合成一個綜合分數。

---

### 🌍 真實故事：GDP 是個說謊精，所以聯合國發明了 HDI

1990 年以前，各國政府比的是 **GDP（國內生產毛額）**。  
GDP 越高 = 國家越好？

不一樣。

```
反例一：沙烏地阿拉伯（1980 年代）
  GDP per capita：非常高
  女性識字率：極低
  平均壽命：中等
  → 有錢，但人民不一定過得好

反例二：古巴（同時期）
  GDP per capita：低
  醫療品質：接近已開發國家水準
  識字率：接近 100%
  → 窮，但人民的「生活品質」某些面向超越很多富裕國家
```

聯合國開發計劃署（UNDP）的 Mahbub ul Haq 和 Amartya Sen 認為：  
**「人類發展」這件事，根本無法用單一數字衡量。**

於是他們在 1990 年創造了 **人類發展指數（HDI）**：

```
「人類發展程度」無法直接量測
        ↓
拆解為三個可觀察的維度：

維度一：健康
  代理指標 → 出生時預期壽命

維度二：教育
  代理指標 → 平均受教年數 + 預期受教年數

維度三：生活水準
  代理指標 → 人均國民所得（GNI per capita，購買力平價校正）

        ↓ 幾何平均合成（每個維度權重相等）

HDI = (健康指數 × 教育指數 × 收入指數)^(1/3)
```

這個複合指標第一次讓世界看到：  
有些國家 GDP 很高，但 HDI 卻排名落後；  
有些國家雖然貧窮，卻在教育和健康上遠超經濟水準。

> **複合指標的精髓：**  
> 當你想衡量的東西「太大、太複雜、太抽象」，  
> 就把它拆開——找到構成它的核心維度，分別量化，再合成。  
> **關鍵在於：你拆的方式，反映了你對「這個東西到底是什麼」的理解。**

---

```
目標：量化「用戶黏性」

原始可觀察指標
├── APP 每日活躍率 (DAU/MAU)
├── 平均每次使用時長
├── 功能點擊頻次
├── 內容分享次數
└── 連續使用天數

         ↓ 加權合成

「用戶黏性指數」= 0.3 × DAU/MAU
               + 0.2 × 正規化(使用時長)
               + 0.2 × 正規化(點擊頻次)
               + 0.15 × 正規化(分享次數)
               + 0.15 × 正規化(連續天數)
```

> 💡 **權重怎麼決定？**  
> (1) 領域專家判斷  
> (2) 主成分分析 (PCA) 讓資料自己說話  
> (3) 從業務目標反推（最重要的行為給最高權重）

---

#### ⚔️ 武器三：質化轉量化 (NLP 情感分析)

**核心思想：** 把人說的話、寫的字，轉換成數字。

---

### ⭐ 真實故事：五星評論背後的數據戰爭

Yelp 每個月有超過 1 億則評論，是美國最大的在地商家評分平台。  
但有一個問題：很多五星評論是假的。

```
Yelp 的工程師面對的挑戰：
  真實評論：「這家餐廳的牛排火候剛好，服務生很親切，
             就是停車有點難找，下次還會來。」
  
  假評論A：「超級好吃！！！強烈推薦！！！必來！！！」
  假評論B：「食物美味，服務一流，環境整潔，物超所值。」

肉眼難以區分，但資料可以。
```

Yelp 的資料科學團隊把「評論的可信度」這個不可量測的概念，  
拆解成數十個可量化的特徵：

```
評論文字特徵：
  ├── 情感極性分數（過於極端 → 可疑）
  ├── 詞彙多樣性（重複詞過多 → 可疑）
  ├── 句子長度分布（太短或太公式化 → 可疑）
  └── 具體細節密度（沒有具體描述 → 可疑）

評論行為特徵：
  ├── 帳號創建至首次評論的時間差（太短 → 可疑）
  ├── 該帳號的評論頻率（突然爆發 → 可疑）
  └── 評論的地理位置與商家距離（從未去過該區 → 可疑）
```

把這些全部合成，得到一個「可信度分數」。  
低於閾值的評論會被自動過濾，不計入星等。

> **這就是質化轉量化的真實應用：**  
> 「這則評論真不真實？」沒有辦法直接回答。  
> 但你可以把「真實性」拆成幾十個可量化的訊號，  
> 讓模型學會辨別。

---

```
用戶評論（質化）              情感分數（量化）
─────────────────────────────────────────────
「這個 APP 超好用，每天都在用！」  →  +0.92
「還可以，不算難用」              →  +0.15
「客服態度很差，不推薦」          →  -0.78
「介面設計跟以前比差好多」        →  -0.45
「超級好吃！！！必來！！！」      →  +0.91（但可信度分數：低 ⚠️）
```

**現代做法：**
```
文字輸入
  ↓
BERT / GPT 類語言模型
  ↓
情感分類（正/負/中）+ 強度分數（-1 到 +1）
  ↓
可以放進你的資料集當作特徵
```

---

> ### ❓ 常見問題 FAQ
>
> **Q：我用了代理指標，但感覺測的不準，怎麼辦？**  
> A：這很正常。代理指標本來就是妥協的產物。重要的是：(1) 使用多個代理指標互相驗證；(2) 在報告中說明這是「近似測量」而非「直接測量」；(3) 持續蒐集真實指標的資料（例如透過問卷或用戶訪談）來校驗你的代理指標。
>
> **Q：情感分析一定準確嗎？**  
> A：不一定，尤其是對「反諷」、「台灣在地俚語」的處理能力有限。務必在你的特定領域進行校驗，不要盲信模型輸出。

# 終點站：你的資料處理 SOP 🏁

---

```
┌─────────────────────────────────────────────────────┐
│           資料從收集到可用的完整流程                  │
└─────────────────────────────────────────────────────┘

  STEP 1: 探索性資料分析 (EDA)
  ─────────────────────────────
  目標：搞清楚你拿到的是什麼東西
  
  ✅ 查看每個欄位的資料型態
  ✅ 畫分布圖，找離群值
  ✅ 計算缺失率：哪些欄位最嚴重？
  ✅ 檢查類別欄位：有沒有奇怪的值（錯字、大小寫不一致）？
  ✅ 相關性矩陣：有沒有高度相關的重複特徵？

  STEP 2: 資料清洗 (Cleaning)
  ─────────────────────────────
  目標：讓資料乾淨、一致、可信
  
  ✅ 統一單位（公斤 vs 磅、台幣 vs 美元）
  ✅ 去除重複樣本 (Deduplication)
  ✅ 處理格式錯誤（日期格式、電話號碼格式）
  ✅ 處理離群值（刪除？Winsorize？保留？）
  ✅ 處理缺失值（依三種機制選擇對應方法）

  STEP 3: 偏誤審查
  ─────────────────────────────
  目標：確認資料代表性
  
  ✅ 核對樣本分布 vs 真實母體分布
  ✅ 檢查標籤一致性（計算 κ 值）
  ✅ 確認是否有系統性排除某些族群

  STEP 4: 敏感度分析
  ─────────────────────────────
  目標：確認你的處理決策不會扭曲結果
  
  ✅ 比較「刪除缺失」vs「填補缺失」後的模型表現
  ✅ 測試不同填補方法對結論的影響
  ✅ 確認代理指標與真實指標的相關係數

  STEP 5: 建立 Data Pipeline 驗證機制
  ─────────────────────────────
  目標：從源頭預防問題
  
  ✅ 定義 Schema（欄位名稱、型態、允許範圍）
  ✅ 建立入資料的自動驗證規則（如 Great Expectations）
  ✅ 設定異常警報（缺失率突然升高 → 自動通知）
```

```
  EDA → 清洗 → 偏誤審查 → 敏感度分析 → Pipeline 驗證
```

## 🎖️ 回到那間小房間

還記得鮑爾排長的故事嗎？

那間神秘小房間的問題，不是裡面的人不夠努力，  
而是他們從來沒有走出去問：**「我們收集的這些資料，真的能回答我們的問題嗎？」**

活動記錄主要來自白天的巡邏回報，夜間的資料幾乎是空白的——  
**這個缺失本身，就是最重要的答案。**

這是 MNAR。  
不是因為夜晚沒有發生任何事，

```
資料缺失的地方，往往藏著最關鍵的真相。
```
