## 🧑‍🏫 教學建議

### 課堂節奏

```
每堂課（3 小時）建議分配：

00:00 - 00:15   複習上週重點 + 作業檢討
00:15 - 00:45   新概念講解（搭配投影片）
00:45 - 01:30   跟做實作（Live Coding）
01:30 - 01:45   休息
01:45 - 02:30   自主練習 + 助教巡迴
02:30 - 02:50   小組討論 + 分享
02:50 - 03:00   下週預告 + 作業說明
```

### 教學原則

1. **先 Demo 再讓學生做**：不要一開始就丟作業
2. **用真實資料**：教學資料集比隨機數字更有感
3. **鼓勵犯錯**：錯誤是最好的學習機會
4. **少講多做**：學生動手的時間 > 老師講課的時間
5. **重複重要觀念**：`groupby`、`merge`、`缺失值處理` 要多次練習

### 常見教學挑戰

| 挑戰 | 建議對策 |
|------|----------|
| 學生程度差異大 | 設計分級作業（基本 + 進階） |
| 學生只會複製貼上 | 要求口頭解釋程式碼邏輯 |
| 環境問題多 | 提供 Google Colab 備案 |
| 不知道何時該用哪個函式 | 準備速查表（附錄 B） |
| 資料清理覺得無聊 | 強調「真實工作 80% 在清資料」 |

## 💭 教學哲學

### Pandas 教學不是教 API

```
❌ 傳統方式：
   「pd.read_csv() 的參數有 filepath, sep, header, index_col, usecols...」

✅ 更好的方式：
   「你有一個 CSV 檔案，裡面的中文都是亂碼，怎麼辦？」
   → 引導學生自己發現需要 encoding 參數
```

### 以「問題」驅動學習

每一章的教學都應該從一個**具體問題**開始：

| 章節 | 問題驅動 |
|------|----------|
| Ch04 EDA | 「這筆資料乾不乾淨？有沒有問題？」 |
| Ch05 清理 | 「有 5% 的訂單沒有地區資訊，怎麼辦？」 |
| Ch07 GroupBy | 「老闆問各地區營收，你怎麼算？」 |
| Ch09 時間序列 | 「這個月比上個月好嗎？」 |

### 培養「資料直覺」

最終目標不是學生會用 Pandas，而是：

1. **能問對問題**：看到資料知道該問什麼
2. **能選對工具**：知道用 groupby 還是 pivot_table
3. **能解讀結果**：數字背後代表什麼商業意義
4. **能溝通發現**：用非技術語言向他人說明
