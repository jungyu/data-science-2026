# Ch01：為什麼要學 Pandas？

> **本章目標**：理解 Pandas 在資料科學生態系中的角色，建立正確的學習心態。

---

## 🧠 先想一個問題

假設你是一家電商的資料分析師，老闆問你：

> 「上個月哪個地區的銷售額最高？退貨率呢？」

你會怎麼做？

- 用 Excel？ → 資料量一大就當機
- 用 SQL？ → 可以查，但不方便做視覺化
- 用純 Python？ → 可以，但你得自己寫一堆迴圈

**Pandas 的答案**：

```python
import pandas as pd

df = pd.read_csv("data/orders.csv")
df.groupby("region")["unit_price"].sum().sort_values(ascending=False)
```

三行程式碼，搞定。

## 🎯 Pandas 幫你解決的三件事

### 1. 讀資料 — 什麼格式都能吃

| 格式 | 函式 | 常見來源 |
|------|------|----------|
| CSV | `pd.read_csv()` | 開放資料、匯出報表 |
| Excel | `pd.read_excel()` | 企業報表、人工整理 |
| JSON | `pd.read_json()` | Web API、NoSQL |
| SQL | `pd.read_sql()` | 關聯式資料庫 |
| HTML | `pd.read_html()` | 網頁表格 |
| Parquet | `pd.read_parquet()` | 大數據平台 |

### 2. 清資料 — 真實世界的資料都是髒的

```
原始資料的真相：
├── 有空值（客戶沒填 email）
├── 有重複（同一筆訂單出現兩次）
├── 有錯誤（年齡 = -5？）
├── 格式不一（日期有的寫 2024/1/1，有的寫 Jan 1, 2024）
└── 編碼問題（中文變亂碼）
```

Pandas 提供了一整套工具來處理這些問題。

### 3. 分析資料 — 從數字到洞察

```python
# 不只是「算平均」，而是「回答問題」
df.groupby("region")["unit_price"].mean()          # 各地區平均客單價
df.pivot_table(values="quantity", index="category") # 各品類銷量
df.resample("M").sum()                              # 月度趨勢
```

## 🆚 為什麼不用 Excel？

| 比較項目 | Excel | Pandas |
|----------|-------|--------|
| 資料量上限 | ~100 萬列 | 記憶體有多大就能多大 |
| 自動化 | 需要 VBA（難維護） | Python 腳本（易維護） |
| 可重現性 | 手動操作，難以追蹤 | 程式碼即文件 |
| 版本控制 | 幾乎不可能 | Git 完美支援 |
| 與 ML 整合 | 困難 | 無縫銜接 scikit-learn |
| 協作 | 檔案傳來傳去 | GitHub / Jupyter |

> **Excel 不是不好**，它很適合快速瀏覽和小規模資料。
> 但當你需要「可重現、可自動化、可擴展」的分析時，Pandas 是更好的選擇。

## 💡 學習心態建議

### ✅ 正確心態

- **「我在學資料思維，不是在背 API」**
- 每次用 Pandas，都在練習：問對問題 → 找到資料 → 寫出分析 → 解釋結果
- 不用一次記住所有函式，用到再查就好

### ❌ 常見誤區

- ~~「我要把所有 method 都背起來」~~ → Pandas 有 200+ 個方法，背不完的
- ~~「我要一次寫出完美的程式」~~ → 先能跑，再優化
- ~~「看影片就等於學會了」~~ → 沒動手寫過就是沒學會

## 🔑 本章重點回顧

1. Pandas 是資料科學的**核心工具**，負責「讀取 → 清理 → 分析」
2. 比 Excel 更適合**大規模、可重現、可自動化**的分析
3. 學 Pandas 的重點不在背 API，而在**建立資料思維**
4. 在 AI 時代，Pandas 是你和資料之間的**翻譯層**
