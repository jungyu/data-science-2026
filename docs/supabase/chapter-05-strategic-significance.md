# 第五章：Supabase 在資料科學實務中的戰略意義

---

## 5.1 能力對比

對資料科學學生來說：

| 只會 Pandas | 會 Supabase |
|-------------|-------------|
| 分析型 | 系統型 |
| 做報告 | 做產品 |
| Notebook | API |
| 無權限 | RLS |
| 無部署 | 雲端化 |

---

## 5.2 三種層次的資料科學家

### 第一層：分析者

- 工具：Pandas、Jupyter Notebook
- 產出：報告、圖表
- 限制：無法部署、無法協作

### 第二層：資料工程師

- 工具：PostgreSQL、SQL、Index
- 產出：結構化資料模型
- 能力：資料建模、查詢優化

### 第三層：產品型資料科學家

- 工具：Supabase、REST API、RLS
- 產出：可部署的資料系統
- 能力：API 設計、權限控制、系統架構

---

## 5.3 職場價值

學生畢業後會：

- 懂資料庫
- 懂權限
- 懂 API
- 懂部署
- 懂 Migration
- 懂版本控制

這些能力比多會一個模型重要。

---

## 5.4 課程章節結構建議（18 小時版本）

### 第一階段（基礎）— 6 小時

1. PostgreSQL 基礎
2. SQL 查詢與 Index
3. JSONB 與資料建模

### 第二階段（Supabase）— 6 小時

4. Supabase 架構
5. Auth + API
6. RLS

### 第三階段（實戰）— 6 小時

7. Python + Supabase
8. 資料科學 API 建立
9. 專案發表

---

## 本章核心觀念

> PostgreSQL 是資料科學的「資料引擎」
> Supabase 是資料科學的「產品化引擎」

> 只會 Pandas 是分析者
> 會 PostgreSQL 是資料工程師
> 會 Supabase 是產品型資料科學家
